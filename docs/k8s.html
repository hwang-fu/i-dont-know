  1. Introduction

Kubernetes Introduction

  1. What Is Kubernetes?

    • Kubernetes (also called K8s) is an open-source system for automating the deployment, scaling, and management of containerized applications.

    • Originally developed by Google, now maintained by CNCF (Cloud Native Computing Foundation).

    • Kubernetes helps you run large-scale distributed applications by managing:
      • containers (Docker, containerd, CRI-O)
      • networking
      • scaling
      • load balancing
      • configuration & secrets
      • storage

    • It is the industry-standard platform for orchestrating production container environments.


  2. Why Kubernetes?

    • Kubernetes provides solutions for problems that arise when running containers at scale:

      • Automatic restarts if containers crash
      • Automated scaling based on CPU / memory / custom metrics
      • Load balancing across multiple containers
      • Zero downtime deployments with rolling updates
      • Self-healing (recreate failed containers automatically)
      • Service discovery and internal DNS
      • Infrastructure abstraction — run anywhere

    • Kubernetes is heavily used in cloud computing and modern DevOps pipelines.


  3. Kubernetes Architecture Overview

    • Kubernetes follows a master-worker architecture:

      • Component Role
      Control Plane (Master Node) Manages the cluster, API, scheduling, health checks
      Worker Nodes Runs applications and containers

    • Main control plane components:
      • kube-apiserver — main entry point (REST API)
      • kube-scheduler — decides where containers run
      • kube-controller-manager — handles scaling, replication, node health
      • etcd — cluster database (stores all cluster state)

    • Worker node components:
      • kubelet — manages containers on the node
      • kube-proxy — networking & load distribution
      • container runtime — Docker, containerd, CRI-O


  4. Key Kubernetes Concepts

    • Core objects you will interact with:

      • K8s Object Description
      Pod Smallest deployable unit; wraps one or more containers
      Deployment Manages replicas of Pods; supports rolling updates
      Service Exposes Pods internally or externally; provides load balancing
      Ingress Routes external HTTP/HTTPS traffic to Services
      ConfigMap Non-secret configuration values
      Secret Base64-encoded sensitive data (passwords, tokens)
      Volume Persistent storage
      Namespace Logical grouping for isolation


  5. Working with Kubernetes: kubectl

    • kubectl is the CLI for Kubernetes.

    • Common commands:
      • kubectl get pods
kubectl get deployments
kubectl get services
kubectl apply -f app.yaml
kubectl delete pod mypod

    • apply is used to create or update Kubernetes resources.


  6. Example: Simple Deployment YAML

    • A minimal deployment file:
      • apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
        - name: myapp
          image: nginx:latest
          ports:
            - containerPort: 80

    • This defines:
      • 3 running replicas
      • using the nginx image
      • exposing port 80


  7. Service YAML Example

    • Expose the Deployment internally:
      • apiVersion: v1
kind: Service
metadata:
  name: myapp-service
spec:
  selector:
    app: myapp
  ports:
    - port: 80
      targetPort: 80
  type: ClusterIP

    • Service types:
      • ClusterIP — default, internal only
      • NodePort — exposes from each node
      • LoadBalancer — integrates with cloud load balancers


  8. Ingress Example

    • Ingress exposes HTTP/HTTPS URLs:
      • apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
spec:
  rules:
    - host: example.com
      http:
        paths:
        - path: /
          pathType: Prefix
          backend:
            service:
              name: myapp-service
              port:
                number: 80

    • Requires an Ingress Controller (NGINX, Traefik, etc.).


  9. Scaling Applications

    • Manual scaling:
      • kubectl scale deployment myapp --replicas=10

    • Autoscaling (HPA):
      • kubectl autoscale deployment myapp --cpu-percent=50 --min=2 --max=10

    • Kubernetes monitors resource usage and adjusts replicas automatically.


  10. Kubernetes Distributions and Platforms

    • You can run Kubernetes almost anywhere:

    • Local Development:
      • Minikube
      • Kind (Kubernetes-in-Docker)
      • MicroK8s

    • Cloud Providers:
      • AWS EKS
      • Google GKE
      • Azure AKS
      • DigitalOcean Kubernetes

    • Enterprise Platforms:
      • OpenShift
      • Rancher
      • VMware Tanzu


  11. Kubernetes Use Cases

    • Kubernetes is used for:
      • microservices architectures
      • stateless + stateful applications
      • CI/CD automation
      • machine learning workloads
      • large-scale web applications
      • hybrid cloud deployments

    • It is now the backbone of modern cloud infrastructure.
