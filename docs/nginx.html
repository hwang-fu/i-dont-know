Nginx for Beginners

  1. What Is Nginx?

    • Nginx (pronounced “engine-x”) is a high-performance web server widely used for:
      • serving static websites
      • acting as a reverse proxy
      • load balancing
      • caching
      • handling HTTPS/SSL

    • Nginx is known for being:
      • very fast
      • lightweight
      • scalable
      • easy to configure

    • It is used by companies like Netflix, Dropbox, GitHub, Airbnb, and many more.


  2. How Nginx Works (Beginner-Friendly Explanation)

    • Nginx uses an event-driven architecture instead of creating a new process for each request.

    • This makes Nginx extremely efficient:
      • handles thousands of connections with low memory usage
      • ideal for high-traffic websites

    • Common roles of Nginx:
      • Web Server: serves HTML, images, CSS, JS
      • Reverse Proxy: forwards requests to backend apps like Flask, Django, Node.js
      • Load Balancer: distributes traffic across multiple servers
      • SSL Terminator: handles HTTPS connections


  3. Installing Nginx

    • On Ubuntu/Debian:
      • sudo apt update
sudo apt install nginx

    • On CentOS/RHEL:
      • sudo yum install nginx

    • Start Nginx:
      • sudo systemctl start nginx
sudo systemctl enable nginx

    • Verify:
      • curl http://localhost


  4. Nginx Configuration File Structure

    • Main config file:
      • /etc/nginx/nginx.conf

    • Server configs (most important):
      • /etc/nginx/sites-available/
/etc/nginx/sites-enabled/

    • Typical workflow:
      • Create a config file in sites-available
      • Enable it by linking into sites-enabled
      • Reload nginx


  5. Basic Nginx Server Block (Virtual Host)

    • Example: Serving a static website
      • server {
    listen 80;
    server_name example.com;

    root /var/www/example;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

    • This configuration:
      • Listens on port 80
      • Serves /var/www/example/index.html
      • Returns 404 if file not found


  6. Serving Static Files

    • Static assets like images, CSS, JS can be served extremely fast by Nginx.

    • Example:
      • location /static/ {
    alias /var/www/example/static/;
}

    • alias is used instead of root when mapping a URL path to a different directory.


  7. Nginx as a Reverse Proxy (for Flask, Django, Node.js, etc.)

    • This is one of the most common use cases:
      • server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

    • What this does:
      • Receives HTTP requests
      • Forwards them to your backend app (Gunicorn, uWSGI, Node.js)
      • Passes user IP and host headers correctly


  8. Enabling HTTPS with Nginx

    • Install Certbot for Let's Encrypt:
      • sudo apt install certbot python3-certbot-nginx

    • Enable HTTPS:
      • sudo certbot --nginx -d yourdomain.com

    • Certbot:
      • configures SSL automatically
      • sets up renewal systemd timers
      • redirects all traffic to HTTPS


  9. Common Nginx Commands

      10. sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl reload nginx
sudo systemctl status nginx

    • Check configuration syntax:
      • sudo nginx -t


  10. Nginx Logging

    • Default logging locations:
      • /var/log/nginx/access.log
/var/log/nginx/error.log

    • Useful for debugging “404 Not Found”, “502 Bad Gateway”, and server errors.


  11. Nginx Security Basics

    • Disable server tokens:
      • server_tokens off;

    • Rate limiting (simple example):
      • limit_req_zone $binary_remote_addr zone=mylimit:10m rate=10r/s;

    • Hide directory listing:
      • autoindex off;

    • Always use HTTPS in production.


  12. Where Nginx Is Used in Real Projects

    • Nginx is often used to:
      • serve front-end (React/Vue) static code
      • proxy requests to backend apps (Flask, Django, Spring, Express)
      • load balance microservices
      • handle SSL termination
      • cache large or frequently used files
      • act as an API gateway

    • Most production systems use Nginx or Apache as their main web entrypoint.
